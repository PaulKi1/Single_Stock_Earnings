# Earnings Options Pipeline


## Requirements

- Python 3.9+ (uses the stdlib `zoneinfo`)
- A running **TWS or IB Gateway** session, logged in with 
API enabled (default ports: 7496 live / 7497 paper)
- An **IBKR Wall Street Horizon** subscription (Modules 01 & 02 event data) —
  see IBKR for current pricing
- A **Databento** API key with OPRA access (Module 03) — see databento.com
- Python packages in `requirements.txt`

## Setup

```bash
# 1. Clone
git clone https://github.com/<your-username>/<your-repo>.git
cd <your-repo>

# 2. Create and activate a virtual environment
python -m venv .venv
# Windows (PowerShell):
.venv\Scripts\Activate.ps1
# macOS / Linux:
# source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Add your Databento key
#    Copy ..env.example to .env and paste your key in.
```

## Running

```bash
python run_earnings_options_pipeline.py
```

Edit the config block at the top of the runner to choose which modules run, the
lookback window, incremental mode, and Databento cost controls. Then launch the
dashboard:

```bash
python earnings_straddle_dashboard.py
```

## Folder layout

```
.
├── run_earnings_options_pipeline.py     # orchestrator
├── earnings_calendar_ibkr_wsh.py        # Module 01
├── underlying_price_ibkr.py             # Module 02
├── databento_option_chain_downloader_optimized.py  # Module 03
├── straddle_price_builder.py            # Module 04
├── earnings_straddle_dashboard.py       # dashboard
├── Tickers.xlsx                         # input: the ticker universe
├── requirements.txt
├── .env.example                         # template for your Databento key
└── data/                                # generated outputs (gitignored)
    ├── 01_earnings_calendar/
    ├── 02_underlying_prices/
    ├── 03_option_chains/
    └── 04_straddles/
```

`data/` is gitignored — it is regenerated by running the pipeline. A fresh clone
will not contain the final workbook, so run the pipeline before the dashboard.

## Notes

- **Costs:** Module 03 reuses its `.dbn.zst` request cache before any Databento
  call, so re-runs of already-downloaded targets are free. New tickers with deep
  chains can cost real money — consider enabling the cost estimate and budget in
  the runner config before a large run.
- **Not financial advice.** This is research tooling.
